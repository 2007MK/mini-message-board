# Mini Message Board

### Project Context
This is a foundational project developed during my initial transition into **Backend Development**. The objective was to move beyond static frontends and master the core mechanics of server-side logic, specifically:
* Handling HTTP GET and POST requests.
* Managing data flow between a client and a Node/Express server.
* Implementing server-side rendering with EJS.

## Features
* **Message Feed:** View a list of all messages with authors and timestamps.
* **Interaction:** Post new messages via a dedicated form.
* **Dynamic Rendering:** Built using EJS (Embedded JavaScript) for server-side templating.
* 
## UI/UX Philosophy
This project intentionally uses **Raw HTML**. There is no CSS framework or complex styling included. The design is kept "bare-bones" to ensure 100% focus on the underlying backend architecture and request-response cycle rather than visual aesthetics.

## 🛠️ Tech Stack
* **Runtime:** Node.js
* **Framework:** Express.js
* **View Engine:** EJS
* **Styling:** None (Raw HTML)
